Observation
The SMOTE Function was used to balance the dataset, however in a real world
scenario we wouldnt have as much frauds compared to none frauds.
We used the XGBoost Model for our project because it was the strongest model.
We used the non-smote version of the model which gave us 0.29% True Negatives
which is a more real world scenario of the amount of frauds.
SMOTE is a great way to solve class imbalance problems however for this project,
the goal was to make our fraud detector similar to a real world scenario.
